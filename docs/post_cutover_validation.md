# Post-Cutover Validation Checklist

Run on the day of cutover (T+0) and again 24 hours later (T+1) and 7 days later (T+7).

## 1. Master Data

- [ ] `SELECT COUNT(*) FROM tabFarm` matches legacy `farms`.
- [ ] `SELECT COUNT(*) FROM tabFarm House` matches legacy `farm_houses`.
- [ ] `SELECT COUNT(*) FROM tabFlock` matches legacy `poultry_flocks`.
- [ ] `SELECT COUNT(*) FROM tabBreed Standard` matches legacy `poultry_breed_standards`.
- [ ] Every active flock has a linked Project, Cost Center, and Warehouse.
- [ ] Every active flock has a `current_count` matching the Postgres delta within +/- 0.5%.

## 2. Daily Operations

- [ ] Last 30 days of daily logs are present and submitted.
- [ ] Total feed intake last 30 days matches legacy within +/- 0.5%.
- [ ] Total mortality last 30 days matches legacy within +/- 0.5%.
- [ ] Each Stock Entry generated by feed/medicine has a posting date that matches the source log date.
- [ ] FCR per active flock matches legacy within +/- 1.0%.

## 3. Health & Treatments

- [ ] All active treatment rounds have linked Medicine Lots and remaining qty matches Stock Ledger.
- [ ] No round has `status = Missed` for which a corresponding event exists in Postgres.
- [ ] Health records last 30 days match legacy.

## 4. Sales / Buying

- [ ] All Poultry Sales Orders submitted in last 30 days have linked Sales Invoices.
- [ ] All Farm Purchases submitted in last 30 days have linked Purchase Invoices.
- [ ] Outstanding receivables sum matches legacy A/R aging within +/- 0.1%.

## 5. Accounting

- [ ] Journal Entries posted for each MORTALITY_LOSS, SLAUGHTER_TRANSFER event in last 30 days.
- [ ] Cost Center per flock has the expected balances (debit / credit columns).
- [ ] Trial balance reconciles end-to-end.

## 6. HR / Payroll

- [ ] Latest Salary Slip period has matching Flock Payroll Allocation.
- [ ] Sum of allocated amounts equals Salary Slip net pay (rounded to 2dp).
- [ ] No employee has overlapping Laborer Assignments > 100% on any day.

## 7. Reports & Dashboards

- [ ] Flock Performance script report renders for at least 5 active flocks.
- [ ] Flock P&L reconciles to General Ledger within +/- 0.5%.
- [ ] Flock Mortality Trend chart shows last 90 days correctly.
- [ ] Treatment Compliance report shows expected counts.
- [ ] Broiler Cycle Report PDF renders without errors.
- [ ] Investor Report PDF renders without errors.

## 8. Permissions / Mobile

- [ ] Each role logs in successfully (Manager, Vet, Laborer, Accountant, Owner).
- [ ] A laborer can ONLY see flocks they are assigned to.
- [ ] `submit_check_in` API returns 200 for an assigned laborer and 403 for a non-assigned user.
- [ ] Daily Log mobile flow works on a real phone.

## 9. Backups & Monitoring

- [ ] Nightly MariaDB dump succeeded (verify file size & integrity).
- [ ] Nightly site files backup succeeded.
- [ ] All workers (default, long, short) are alive in Render dashboard.
- [ ] Scheduler last-run timestamp is < 6 minutes old.
- [ ] Redis cache hit rate > 70%.
